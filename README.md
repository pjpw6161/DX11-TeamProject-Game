# 🎪 Popup Store Simulator (3D 팝업스토어 경영 시뮬레이션)

![Game Banner Placeholder](https://dummyimage.com/800x400/2b2b2b/ffffff&text=Popup+Store+Simulator)
> **"살아 숨 쉬는 도시 속에서 나만의 팝업스토어를 성공적으로 운영하라!"** > 활기찬 군중 AI, 자율주행 교통 시스템, 그리고 다이내믹한 카메라 뷰를 제공하는 3D 타이쿤 시뮬레이션 게임입니다.

<br>

## 🎮 프로젝트 개요
* **개발 엔진:** Unity 3D (버전: 6000.3.9f1)
* **주요 기술:** C#, NavMesh, Cinemachine, Object Pooling, DOTween
* **파트:**: Infra,Client(Unity)
* **플랫폼:** WebGL / PC

<br>

## ✨ 핵심 기능 및 기술적 성과 (Key Features & Tech Specs)

포트폴리오의 완성도와 성능을 높이기 위해 **게임 최적화**와 **자연스러운 AI 상호작용**에 집중했습니다.

### 1. 🚶‍♂️ 대규모 군중 AI 및 오브젝트 풀링 (Crowd AI & Object Pool)
수많은 방문객이 맵을 돌아다녀도 프레임 드랍이 발생하지 않도록 설계했습니다.
* **오브젝트 풀링(Object Pooling) 적용:** 게임 시작 시 설정된 최대 인원(예: 100명)만 미리 생성(Instantiate)하고, 이후에는 풀에서 꺼내어 재활용함으로써 GC(가비지 컬렉터) 스파이크를 원천 차단했습니다.
* **자연스러운 군중 흐름:** `NavMeshAgent`의 `Obstacle Avoidance`와 우선순위(Priority) 랜덤 배정을 통해, 좁은 길에서 NPC들이 겹치거나 뭉치는(Mosh Pit) 데드락 현상을 방지했습니다.
* **충돌 연산 최소화:** Physics Layer Matrix를 조작하여 NPC 간의 불필요한 물리 충돌(Collider) 연산을 제거했습니다.

### 2. 🚗 Raycast & NonAlloc 기반 자율주행 교통망
도시의 리얼리티를 살리기 위해, 연산이 무거운 `NavMesh Obstacle` 대신 가벼운 물리 센서를 장착한 자율주행 차량을 구현했습니다.
* **메모리 최적화:** 매 프레임 발생하는 쓰레기 메모리를 막기 위해 `Physics.OverlapBoxNonAlloc`을 사용하여 차량 전방의 장애물을 감지합니다.
* **거리 연산 최적화:** 무거운 루트 연산이 들어가는 `Vector3.Distance` 대신 `sqrMagnitude`를 사용하여 성능을 3배 이상 끌어올렸습니다.
* **교착 상태(Deadlock) 해결:** 교차로에서 차량들이 마주쳤을 때 영원히 멈추는 현상을 막기 위해, 각 차량에 랜덤한 '인내심(Patience) 타이머'를 부여하여 자연스럽게 양보하고 출발하는 로직을 구현했습니다.
* **배경 디오라마 (Ambient Life):** 고속도로와 철도를 달리는 배경용 차량/기차는 `Update` 문에서 `localPosition`만 조작하고 `Coroutine`으로 재배치하여, 연산 비용을 최소화한 무한 루프 트래픽을 구현했습니다.

### 3. 🎥 Cinemachine 기반 하이브리드 카메라 시스템
플레이어가 도시 전체를 관망하거나 특정 구역에 집중할 수 있도록 하이브리드 시점을 제공합니다.
* **8구역 고정 뷰 & 타겟 추적:** `CinemachineCamera`를 활용해 8개의 팝업스토어 구역을 즉시 비추고, '도둑 발생' 같은 특정 이벤트 시 해당 대상을 자동으로 줌인(Follow & LookAt)합니다.
* **자유 비행 모드 (God Mode):** `Spacebar`를 누르면 시네머신의 타겟 고정을 해제하고, WASD와 마우스 휠을 이용해 맵 전체를 자유롭게 날아다니는 탑뷰 모드로 전환됩니다.
* **가두리 양식장 (Mathf.Clamp):** 플레이어가 맵 밖으로 이탈하지 않도록 `Mathf.Clamp`를 적용하여 카메라의 X, Y, Z 이동 범위를 안전하게 제한했습니다.

### 4. 🎪 DOTween 연출 & 시스템 모듈화
* 건물이 스폰될 때 딱딱하게 나타나지 않고, `DOTween`의 `OutBounce` 이즈(Ease)를 활용해 '띠용~' 하고 등장하는 생동감 있는 애니메이션을 적용했습니다.
* 스폰 사운드, 파티클 이펙트의 크기 및 높이를 부모 객체의 스케일에 맞춰 실시간(Update)으로 동기화하는 로직을 구축했습니다.

## 🕹️ 조작 방법 (Controls)
* **마우스 클릭:** UI 상호작용 및 구역 선택
* **Spacebar:** 자유 비행 카메라(God Mode) ON / OFF
* **W, A, S, D:** 자유 비행 모드 시 카메라 전후좌우 이동
* **마우스 휠:** 자유 비행 모드 시 줌인 / 줌아웃

## 💡 개발 회고 (Retrospective)
이 프로젝트를 진행하면서 가장 큰 도전은 **"오브젝트가 많아질 때 프레임이 떨어지는 현상"**을 해결하는 것이었습니다. 단순히 프리팹을 복제(Instantiate)하는 것을 넘어, Object Pooling을 도입하고, 물리 연산(OverlapBox)에 NonAlloc 방식을 적용하며 메모리 구조와 유니티 엔진의 최적화 원리를 깊이 이해할 수 있었습니다. 또한, Cinemachine과 DOTween 같은 에셋을 활용해 코드의 복잡도는 낮추면서 게임의 '보는 맛'을 퀄리티 있게 끌어올리는 경험을 했습니다.
